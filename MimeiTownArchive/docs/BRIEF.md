# 未明町アーカイブ Brief

## One-Line Pitch

亡くなった祖母の遺品整理で見つかった一冊の調査ノートをきっかけに、
購入者自身が未明町の公開記録をたどり、20年前の失踪事件と町の封印された歴史を解く。

## Product Goal

Boothで販売できる、5から8時間級の大型Web謎解き作品を作る。
購入者はZIPを展開し、`start.html` から調査を始める。ネット接続なしでも基本進行できるよう、
作品内の全サイトをローカルHTMLとして同梱する。

## Chosen Direction

A案: 地方自治体・新聞・学校サイトが混ざる静かな実在感。

この作品では、謎解き用の派手なUIやホラー演出を前面に出さない。
各サイトは「本当に存在する普通のページ」に見えることを優先する。
不穏さは、文章の齟齬、記録の欠落、日付の不一致、リンク切れに見える隠し導線で出す。

## World

未明町は、かつて灯台建設計画と港湾整備で栄えた小さな海沿いの町。
20年前、県立未明高校の生徒「雨宮澪」が、地域史研究の発表前日に失踪した。
公式には家出として処理されたが、町の複数の記録には、同じ夜にだけ現れる
「灯台のない灯」という言葉が残っている。

## Player Role

プレイヤーは「謎解き参加者」ではなく、亡くなった祖母の遺品から調査ノートを受け取った人物。
祖母は未明町出身の元図書館司書で、晩年まで雨宮澪の失踪事件を調べていた。
同梱された手紙には、こう書かれている。

> もしこの町の記録がまだ読めるなら、澪さんが最後に見た灯りを探してください。

この依頼が、プレイヤーが調査に入る物語上の理由になる。

## Experience Pillars

1. 普通のWebサイトを調べている感覚
2. 検索・照合・日付比較による発見
3. 一見無関係な地域情報がつながる快感
4. 章ごとに解像度が上がる失踪事件
5. 販売作品として破綻しないヒント・復帰導線

## Interactivity

Full interactive build.

- Search-like page
- Investigation notebook
- Keyword entry
- Password and code checks
- Local progress save
- Staged hint pages
- Multiple endings or ending variations
- Offline-first static implementation

## Success Criteria

- プレイヤーが最初の5分で「なぜ調査するのか」を理解できる。
- どのページも、謎解き用の偽物ではなく普通のサイトとして読める。
- 重要な発見は最低2つの情報源で裏取りできる。
- 章ごとに明確な達成感がある。
- Booth購入者が迷わず開始・再開・ヒント参照できる。
